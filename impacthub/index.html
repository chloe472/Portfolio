Focus

Volunteer Management for Non-profits

The Challenge

ImpactHub focused on the operational complexity of coordinating volunteers, coordinators, and admins in a non-profit environment. The product needed clear roles, reliable task flows, and a structure that could be understood by both stakeholders and future builders.

Analysis Priorities

Role clarity: Define separate journeys for volunteers, coordinators, and admins.

System behavior: Document use cases and activity flows for core operations.

Information structure: Model the domain so data relationships were explicit.

Interface direction: Translate requirements into wireframes for review.

My Solution

I conducted requirements analysis and system design for ImpactHub, creating wireframes for volunteer, coordinator, and admin portals alongside detailed use case, domain model, and activity diagrams.

The result was a structured requirements package that connected stakeholder needs to system behavior and interface decisions.
